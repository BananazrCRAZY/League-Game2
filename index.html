Walls

by: Brandon Hatamoto

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I chose to make the game Walls because it's a simple game, but it can be a fun endless game. I've seen many versions of Walls in other games like Minecraft and there was even a TV show that was similar to Walls. Walls is simple and inspiration for this game came easily to me.

2. What I Learned

The game was just too slow, so I sped it up. Also, the background of the game was too similar color to the actually walls which make the game hard. There was an issue that the rate at which walls spawn is different for some people than it is for me.

3. Architecture of My Game

I have many object classes for the game, and there are many components to the object class. The components for the object class would determine the shape and speed of the object, and depending on the object's name, the object may get a special design. Also, the object manager class controls the rate at which walls spawn and the speed of the walls. The game panel controls all the scenes contains the controls for the player.

4. Technical Implementation

I implemented mouse listener so if you click on the text it'll take the player there, but there is also key listener if the player just wants to play with their keyboard. Also, one part of the game I'm proud of is that the player can pause the game by simply clicking "Q" and later hitting "back to game".

5. Most Challenging Part

The most difficult part of this project was trying to figure out how I should get the objects to have png's. Eventually I added a name part to my parent object class and the name would determine which png the object took.

6. Future Fixes and Updates

I would definitely try to simplify my code and stop creating so many objects.
